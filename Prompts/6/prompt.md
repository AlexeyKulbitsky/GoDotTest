The menu should display two options: Play and Exit. Play loads and starts the gameplay scene. Exit closes the application. The menu should be clean and centered on screen — no elaborate styling needed, just readable text and functional buttons.

Pressing Escape a second time while the pause menu is open should behave the same as clicking Resume — it closes the menu and unpauses the game.

The menu itself must be exempt from the pause so its buttons remain interactive. Make sure the power-up timer, if it happens to be running when the player pauses, resumes correctly from where it left off rather than being reset or lost.
The distinction between main menu mode and pause menu mode should be handled within a single menu.gd script using a flag or an exported variable that controls which buttons are visible and what each button does. There should not be two separate menu scenes for the two contexts.
