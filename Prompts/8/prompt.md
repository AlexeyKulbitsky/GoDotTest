Take the existing project as your starting point. I want you to introduce a health system for the player and upgrade the HUD to reflect it. The concept is straightforward - the player has 3 lives, losing all of them ends the game and shows a game over screen. Let me go through each part in detail. The player should have a maximum of 3 lives tracked by the game manager since lives need to persist correctly across level transitions the same way the score does. The game manager should expose methods for losing a life, restoring a life, and reading the current life count. No other script should hold its own local life variable - everything goes through the game manager just like score. Every time the player dies - from falling, enemy contact, or projectile hit - one life should be deducted and the respawn flow should execute as normal as long as at least one life remains. When the last life is lost the respawn should not happen. Instead the game should transition to a game over screen. The game over screen should be a separate scene saved as game_over.tscn with its script game_over.gd. It should display a clear game over message and player's final score. It should offer two options - Play Again and Exit. Play Again resets both the score and the lives in the game manager back to their starting values and loads level 1 from the beginning. Exit closes the application. The game over screen should not be reachable from the main menu and should only appear as a result of running out of lives during a game session. The power-up collectible should restore one life when collected if the player is hot already at maximum lives. This gives the power-up an additional strategic value beyond the speed and jump boost. If the player is already at 3 lives the life restore simply does not happen - no error, no message, just a silent no-op. On the HUD side, the existing score label should remain but the UI should now also display the player's current life count. Represent lives as a row of simple colored squares or circles in the top right corner of the screen - one shape per life, shapes disappear as lives are lost and reappear if a life is restored by a power-up. The life indicators should update in real time whenever the life count changes. Exted ui.gd and ui.tscn to accommodate this rather than creating a separate UI scene for the lives display. The life count should reset to 3 when the player clicks Play Again from the game over screen or when the player clicks Restart from the pause menu. Starting a new session should always begin with a full set of lives. Transitioning between levels should not reset the life count - whatever lives the player had at the end of level 1 carry into level 2 the same way the score does.
