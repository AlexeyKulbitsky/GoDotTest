Hey! This is another follow-up task for the platformer prototype. It should take around 1 to 1.5 hours. Same standards as always — clean code, clear naming, no debug leftovers.
Take the existing project as your starting point. I want you to implement a basic player death and respawn system. There are three ways the player can die: falling below the level, touching an enemy directly, and getting hit by a projectile. In all three cases the result is the same — the player resets to a designated spawn point and the game continues without reloading the entire scene.
Start by adding a spawn point to the level. It should be a simple Node2D with no visuals, saved as its own scene spawn_point.tscn and placed in platformer.tscn just above the ground at the position where the player should appear at the start of the game and after every death. The player's initial position in the scene should match the spawn point — there should be a single source of truth for where the player starts.
The fall death should be handled by defining a death zone threshold — a Y coordinate value below which the player is considered to have fallen out of the level. When the player's vertical position exceeds this threshold, death is triggered. This threshold should be tunable from the editor so it can be adjusted if the level layout changes. A KillZone using an Area2D stretched across the bottom of the level is also an acceptable approach if you prefer that over a coordinate check.
For enemy contact death, the existing placeholder print statements from the previous task should now be replaced with an actual death trigger. Both the chaser enemy body contact and the projectile hit should call the same death handling logic rather than duplicating it.
The death and respawn logic itself should live in player.gd. When death is triggered from any source, the player should be moved back to the spawn point position, its velocity should be fully reset to zero, and the jump counter should be reset to its maximum value so the player is in a clean state upon respawning. The score should not reset on death — the player keeps whatever they collected before dying.
The spawn point position should be read from the scene at runtime rather than hardcoded in the player script. The player should look up the spawn point node by name or group and read its position when respawning, so moving the spawn point in the editor automatically affects where the player respawns without any script changes.
Add the spawn point instance to platformer.tscn, make sure the player starts on top of it, and verify that all three death conditions correctly trigger the respawn. Name the spawn point scene spawn_point.tscn — no script is needed on it unless you find a reason to add one.
